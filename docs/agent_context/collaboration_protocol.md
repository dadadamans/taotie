# 代理协作协议

## 角色

Planner/Reviewer 代理：

- 拥有架构方向、任务分解和质量门。
- 在实现开始之前编写任务计划。
- 审查实现摘要、差异、测试和文档。
- 编写带有以下决策之一的审查：`APPROVED`、`CHANGES_REQUESTED`、`BLOCKED` 或 `NEEDS_HUMAN`。

实现代理：

- 读取共享项目上下文和当前任务计划。
- 仅实现要求的范围。
- 在可行时运行要求的验证命令。
- 在更改后编写实现摘要。
- 不要默默扩展架构或更改不相关的模块。

## 工作流

1. Planner 编写或更新项目上下文和路线图。
2. Planner 编写当前任务计划。
3. 实现代理实现任务。
4. 实现代理编写任务摘要。
5. Reviewer 审查代码、文档和验证输出。
6. Reviewer 编写决策。
7. 编排器、用户或下一个代理遵循决策。

## 决策

- `APPROVED`：进入下一个任务或最终交付。
- `CHANGES_REQUESTED`：修订同一任务。
- `BLOCKED`：暂停直到阻塞被解决。
- `NEEDS_HUMAN`：暂停等待人工批准或指示。

## 工程规则

- 除非计划明确更改，否则保持项目架构不变。
- 保持任务更改范围狭窄。
- 在可行时为新行为添加针对性的测试。
- 不要隐藏失败的验证命令。
- 不要撤销不相关的用户或代理更改。
- 当架构或行为发生变化时，保持持久化文档最新。
